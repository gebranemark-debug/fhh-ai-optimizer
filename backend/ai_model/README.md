# FHH AI Model Layer

Predictive-maintenance ML + FastAPI service. Sits on top of Prompt 1
(PostgreSQL business objects) and Prompt 2 (TimescaleDB sensor + ETL),
and exposes the contract-shaped HTTP API the Streamlit dashboard
consumes.

Endpoint shapes follow `docs/API_CONTRACT-2.md` v1.1 — every key, every
enum value, every error code is verbatim from the contract.

## Files

| File | Purpose |
|---|---|
| `train_model.py` | Trains the Isolation Forest + RandomForestClassifier from `features.parquet` and persists `.pkl` artifacts plus a feature schema and metrics file. |
| `predict.py` | Inference helpers: `detect_anomaly`, `predict_failure_probability`, `get_recommended_action`. Also exposes the locked 5-tier threshold mapping. |
| `api.py` | FastAPI app. All endpoints conform to the API contract; `/health` is an alias of `/machines/{id}` so the build guide's wording is also satisfied. |
| `artifacts/` | Generated by `train_model.py`. Contains `failure_probability_rf.pkl`, `anomaly_isoforest.pkl`, `feature_schema.json`, `metrics.json`. |

## Models

### Failure-probability classifier (RandomForest)
Trained on `target_failure_within_72h` from the ETL output. Output is a
**continuous probability** in `[0.0, 1.0]` via `predict_proba`, never a
hard yes/no — exactly as the build guide specifies. Default
hyperparameters: 300 trees, `class_weight='balanced'` (because the
positive class is ~1% of rows), `random_state=42` for reproducibility.

### Anomaly detector (IsolationForest)
Trained unsupervised on the full feature matrix. Scores get rescaled
to a 0-100 anomaly index using min/max captured at training time, and
the calibration values land in `feature_schema.json`.

## Threshold mapping (locked — do not modify without bumping the build guide)

| Probability | Tier | Action |
|---|---|---|
| `>= 90%` | **critical** | CRITICAL: Stop line immediately. Replace component now. |
| `75–89%` | **high** | HIGH: Schedule maintenance within 24 hours. |
| `50–74%` | **medium** | MEDIUM: Plan maintenance within 7 days. |
| `25–49%` | **low** | LOW: Monitor closely. Increase inspection frequency. |
| `< 25%` | **normal** | NORMAL: Continue routine operations. |

The 5-tier scale collapses to the API contract's 4-tier `risk_tier` for
the response payloads via `predict.tier_to_api_risk_tier`:

```
critical, high  -> "critical"
medium          -> "warning"
low             -> "watch"
normal          -> "healthy"
```

This keeps the contract enums clean while preserving the build guide's
finer 5-tier action language inside `predict.get_recommended_action`.

## Endpoints

All match `docs/API_CONTRACT-2.md` shapes:

| Method | Path | Returns |
|---|---|---|
| `GET` | `/machines` | `{ machines: [...], total }` |
| `GET` | `/machines/{machine_id}` | Machine object (with `risk_score` / `risk_tier` populated by the model) |
| `GET` | `/machines/{machine_id}/health` | Alias of the line above (build-guide naming) |
| `GET` | `/machines/{machine_id}/risk-score` | `{ machine_id, score, tier, highest_risk_component_id, predicted_failure_window_hours, last_updated }` |
| `GET` | `/machines/{machine_id}/predictions` | `{ machine_id, predictions[6], generated_at }` |
| `GET` | `/alerts` | `{ alerts[], total, counts_by_tier }` |
| `GET` | `/alerts/{alert_id}` | Alert object |
| `GET` | `/kpis/overview` | Hero strip data |

Errors use the contract's standard envelope:

```json
{ "error": { "code": "machine_not_found", "message": "...", "status": 404 } }
```

## Setup

### Prereqs

- Prompt 1 already loaded in Supabase (machines, components, etc.).
- Prompt 2's `features.parquet` already generated (in-memory mode is
  fine):
  ```bash
  python backend/timescale/etl.py --in-memory --out backend/timescale/features.parquet
  ```
- `pip install -r requirements.txt` (adds scikit-learn, joblib, fastapi,
  uvicorn — already in the repo).

### 1. Train the models

```bash
python backend/ai_model/train_model.py
```

Reads `backend/timescale/features.parquet`, writes:

```
backend/ai_model/artifacts/
  failure_probability_rf.pkl
  anomaly_isoforest.pkl
  feature_schema.json
  metrics.json
```

Run summary you'll see:

```
[train] feature dataset: 17,280 rows × 43 columns
[train] usable rows after NaN drop: <large number>
[train] class balance: positive=<n>, negative=<n>, positive_rate=~1.26%
[train]   ROC-AUC      = 0.99+
[train]   PR-AUC       = 0.7+
[train] mean anomaly_score(pos) - mean anomaly_score(neg) = >0
```

ROC-AUC near 1.0 is normal here because the labeled failures sit at the
end of an 18-day vibration ramp — the signal is strong.

### 2. Smoke-test inference

```bash
python backend/ai_model/predict.py al-nakheel
```

Prints the anomaly score, failure probability, recommended action, and
the API-contract `risk_tier` for one machine.

### 3. Run the API

```bash
uvicorn backend.ai_model.api:app --reload --port 8000
# then in another terminal:
curl -s http://localhost:8000/machines/al-nakheel | jq
curl -s http://localhost:8000/machines/al-nakheel/predictions | jq
curl -s http://localhost:8000/alerts | jq
```

### 4. Streamlit page

`app.py` has a new **🛠️ Predictive Maintenance** page that calls
`api.py`'s helpers directly (no HTTP round-trip required) and shows:

- Fleet KPI strip
- Per-(machine, component) failure-probability table, color-coded by
  tier (red 90%+, orange 75–89%, yellow 50–74%, green 25–49%, blue <25%)
- Per-machine summary cards
- Active alerts list
- Anomaly score per machine

Run with:

```bash
streamlit run app.py
```

## Configuration

| Env var | Purpose | Default |
|---|---|---|
| `FHH_AI_ARTIFACT_DIR` | Where `predict.py` looks for `.pkl` files | `backend/ai_model/artifacts` |
| `FHH_AI_FEATURES`     | Where `predict.py` reads features from at inference time | `backend/timescale/features.parquet` |
| `DATABASE_URL`        | Supabase Postgres URL (used by upstream layers; not strictly required by `api.py` since machine metadata is constant) | unset |
